Redis 的 AOF 和 RDB 持久化有什么区别？

生成答案

RDB 是某一时刻的数据快照，文件紧凑，适合备份和快速恢复；AOF 以追加日志记录写命令，通常能提供更好的数据安全性，可以按 always / everysec / no 配置刷盘策略。生产中可以同时开启 RDB 和 AOF，在恢复速度和数据安全之间取得平衡 [1]。

最终 RAG 流程

1. Query Rewriting “持久化”扩展为 persistence、RDB、AOF、snapshot、append only file。
2. Hybrid Retrieval 向量相似度负责语义匹配，BM25 强化 AOF、RDB 等精确术语。
3. Reranking 根据术语覆盖和标题覆盖二次排序，让 redis:persistence 排到最前。
4. Grounded Answer 只基于 Top-k 片段生成，答案必须带来源引用。
5. Citation Check 检查 [1] 是否存在，并验证答案与引用上下文有词项重叠。
6. 可解释输出 同时展示 score、vector_score、bm25_score、rerank_score 和来源 URL。
